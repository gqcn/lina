# v0.8.0 定时任务功能

## 变更概述

实现完整的定时任务管理功能，支持任务的增删改查、启用/禁用、执行日志记录，以及基于分布式锁的单例执行策略。

## 业务价值

- **自动化运维**：支持定期执行系统维护任务（如会话清理、数据备份等）
- **灵活调度**：通过 Cron 表达式实现灵活的任务调度策略
- **可靠执行**：基于分布式锁确保单例任务不会重复执行
- **可观测性**：完整的执行日志记录，便于问题排查和性能分析

## 核心功能

### 1. 任务管理
- 创建、编辑、删除定时任务
- 启用/禁用任务
- 手动执行一次任务
- 任务分组管理（系统分组 + 自定义分组）

### 2. 执行策略
- **Cron 调度**：支持标准 Cron 表达式
- **单例执行**：基于分布式锁，同一时刻只有一个实例执行
- **并行执行**：允许多个实例同时执行
- **次数限制**：支持设置最大执行次数，达到后自动停止

### 3. 系统任务
- 系统分组下的内置任务不可删除
- 任务指令显示为 `<函数名>`，不可修改
- 可修改任务名称、描述、Cron 表达式
- 初始化时自动创建：
  - 会话清理任务：每小时清理过期会话
  - 服务器监控任务：每分钟采集服务器指标

### 4. 执行日志
- 记录每次执行的开始/结束时间
- 记录执行耗时和状态（成功/失败）
- 记录错误信息（如果失败）
- 支持按任务名称、状态、时间范围查询

## 技术方案

### 数据库设计
- `sys_job`：定时任务表
- `sys_job_log`：执行日志表
- `sys_locker`：分布式锁表

### 后端实现
- 使用 GoFrame 的 `gcron` 组件进行任务调度
- 实现 `locker` 服务提供分布式锁能力
- 实现 `job` 服务管理任务的 CRUD 和执行逻辑
- 扩展 `cron` 服务支持动态任务注册

### 前端实现
- 任务列表页：VXE-Grid 表格 + 搜索条件
- 任务表单：Vben Form + Modal
- 执行日志页：VXE-Grid 表格 + 详情抽屉

## 约束与限制

1. **安全性**：自定义任务执行 shell 命令存在安全风险，需要严格的权限控制
2. **性能**：大量并行任务可能导致资源耗尽，建议控制并发数
3. **可靠性**：分布式锁依赖数据库，数据库故障会影响任务执行
4. **兼容性**：系统任务的函数签名必须统一为 `func(ctx context.Context) error`

## 后续优化方向

- 任务执行失败告警机制
- 命令白名单或沙箱机制
- 任务执行并发数限制

- 任务执行统计和可视化
